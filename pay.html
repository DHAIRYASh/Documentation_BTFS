Payment Form

The PaymentForm component is a form used for capturing payment information. It includes input fields for the cardholder's name, card number, expiration date, and CVV number.

Dependencies

  • React
  • useState from React
  • './PaymentForm.css

USAGE

import React, { useState } from 'react'; import './PaymentForm.css'; export default function PaymentForm({ onChange }) { // Component implementation }

PROPS

onChange: A function passed as a prop to the PaymentForm component. It is called whenever the value of any input field in the form changes. The function takes two arguments: the name of the field that changed, and the new value of the field.

STATE

  • cardNumber: A state variable that holds the value of the card number input field.
  • expDate: A state variable that holds the value of the expiration date input field.
  • cvv: A state variable that holds the value of the CVV number input field.
  • cardName: A state variable that holds the value of the cardholder's name input field.

Methods

  • handleCardNumberChange: A function called whenever the card number input field value changes. It formats the input and updates state.
  • handleCardNameChange: A function called whenever the cardholder's name input field value changes. It formats the input and updates state.
  • handleCvvChange: A function called whenever the CVV number input field value changes. It formats the input and updates state.
  • handleExpDateChange: A function called whenever the expiration date input field value changes. It formats the input and updates state.

Rendered Output

The PaymentForm component renders a div element with the class name "payment-form". Inside the div, it renders:

  • A h2 element with the class name "form-title" and the text "Payment method".
  • A div element with the class name "grid".

Inside the grid div, there are four div elements with the class name "grid-item" representing the input fields:

  • Cardholder's Name Input Field: An input of type "text" with id "cardName", name "cardName", placeholder "Name on Card", value bound to cardName state variable, autoComplete set to "cc-name", and an onChange event listener calling handleCardNameChange.
  • Card Number Input Field: An input similar to the cardholder's name input field, but with id "cardNumber", name "cardNumber", placeholder "Card Number", autoComplete set to "cc-number", and onChange event listener calling handleCardNumberChange.
  • Expiration Date Input Field: An input similar to the previous input fields, but with id "expDate", name "expDate", placeholder "MM/YY", autoComplete set to "cc-exp", and onChange event listener calling handleExpDateChange.
  • CVV Number Input Field: An input similar to the previous input fields, but with id "cvv", name "cvv", placeholder "CVV", autoComplete set to "cc-csc", and onChange event listener calling handleCvvChange. Includes a small element with the text "Last three digits on signature strip".

The PaymentForm component relies on the styles defined in the 'PaymentForm.css' file. The specific styles are not provided in this documentation.
